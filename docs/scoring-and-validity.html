Scoring & Validity

점수 체계, 유형 판정, 이론적 근거, 검증 상태 — 마지막 갱신: 2026-03-25

1. 테스트 개요

항목내용
형식15문항, 4지선다, 강제선택 (오답 없음)
영역연구/데이터 5문항 · 행정/계약 5문항 · 관계/소통 5문항
측정 축3축 양극 (원칙↔유연, 투명↔신중, 독립↔협력)
유형2 × 2 × 2 = 8가지
소요 시간약 3분
포지셔닝자기발견/교육 보조 도구 (공식 평가 아님)

2. 3축 정의

축 1
원칙 ↔ 유연
+1 원칙: 규정/절차/기준을 우선하는 행위
-1 유연: 상황/맥락/효율을 고려하는 행위
축 2
투명 ↔ 신중
+1 투명: 정보를 즉시 공유/보고/공개하는 행위
-1 신중: 먼저 확인/파악한 뒤 판단하는 행위
축 3
독립 ↔ 협력
+1 독립: 혼자 판단하고 실행하는 행위
-1 협력: 타인과 함께 판단/행동하는 행위

3. 점수 매핑 로직

작동 방식

1
선택지마다 0~2개 축에 +1 또는 -1 배정
60개 선택지(15문항 × 4) 모두 1~2개 축에 점수 배정. 3개 축 전부에 점수가 있는 선택지는 없음
2
15문항 응답 후 축별 합산
각 축 독립 합산. 예: 원칙 +3, 투명 -2, 독립 +1
3
양수/음수로 방향 판정 → 8유형
원칙(+) × 투명(+) × 독립(+) = "소신 수호자" 같은 조합

예시: Q4 (연구 중간보고 지연)

"일부 실험이 예상대로 안 돼서 당초 계획보다 진도가 늦다."

선택지원칙↔유연투명↔신중독립↔협력
A. 솔직하게 보고한다 투명 +1
B. 보고 시점에 정리해서 말한다 유연 -1 신중 -1
C. 동료에게 먼저 조언을 구한다 신중 -1 협력 -1
D. 방법을 바꿔서라도 결과를 낸다 유연 -1 독립 +1
같은 상황에서 어떤 행동을 선택하느냐에 따라 서로 다른 축의 점수가 반응합니다. 질문 텍스트 자체가 아니라, 선택지 뒤의 점수 매핑이 측정하는 것입니다.

축별 +1/-1 행위 패턴

점수대표 행위
원칙 +1규정/절차를 따른다, 기준표를 만든다, 반납한다, 공식 절차를 밟는다
유연 -1상황에 맞게 조정한다, 관행을 참고한다, 효율을 우선한다, 빠르게 처리한다
투명 +1즉시 보고한다, 문서로 남긴다, 솔직하게 논의한다, 공식 채널에 문의한다
신중 -1먼저 파악한다, 확인해본다, 취합해본다, 조언을 구한 뒤 판단한다
독립 +1혼자 판단한다, 스스로 기준을 세운다, 주도적으로 실행한다
협력 -1팀장에게 맡긴다, 함께 정한다, 동료와 상의한다, 팀 회의에 올린다

6. 이론적 근거

축참고 이론/개념출처
원칙↔유연 도덕 판단의 의무론(deontology) vs 결과론(consequentialism) Kohlberg 도덕 발달 이론, Rest의 DIT
투명↔신중 내부고발(whistleblowing) 의사결정의 "즉시 보고 vs 관망" 패턴 Miceli & Near (1992), 조직 침묵 연구
독립↔협력 조직 행동론의 개인주의-집단주의 차원 Hofstede 문화 차원 이론, Triandis (1995)
설계 방식: 위 이론에서 개념만 차용하여 공공 연구기관 맥락에 맞게 재구성했습니다. 학술적으로 검증된 표준 척도(validated scale)를 직접 사용한 것이 아닙니다. 문항은 연구비 집행, 물품 구매, 용역 계약, 외부 협력 등 실무 상황을 기반으로 자체 개발했습니다.

포지셔닝

이 테스트는 "과학적 진단 도구"가 아닌 "자기발견 및 교육 보조 도구"입니다.

  • 16Personalities(MBTI 기반), StrengthsFinder 등 대중적 성향 테스트와 유사한 위치
  • 결과 화면에 "공식 평가와 무관", "부담 없이 즐기는 셀프 테스트" 명시
  • AI 조언은 "옳고 그름 판단 금지" 원칙으로 프레이밍

7. 검증 상태

항목상태비고
내용 타당도 (Content Validity) 부분 완료 문항 매핑 검토 + 사회적 바람직성 매칭 + 투명/신중 균형 재조정
구성 타당도 (Construct Validity) 미검증 요인분석 등 통계적 검증 미실시
신뢰도 (Reliability) 미검증 Cronbach's alpha 등 미측정
파일럿 테스트 미실시 배포 후 진행 예정
외적 타당도 해당 없음 자기발견 도구 목적, 인사 평가 용도 아님

8. 알려진 한계 & 개선 계획

원칙+독립 동시부여

15문항 중 9문항의 A선택지가 {principle:+1, independence:+1}. 두 축이 독립적으로 측정되지 않아, "원칙적이지만 협력적인 사람"이 제대로 구분되기 어려움.

계획 Q16~Q18 추가: 원칙+협력, 유연+독립 조합 문항

투명↔신중 축 잔여 불균형

투명 17회 vs 신중 14회 (차이 3). 허용 범위이지만 완전 균형은 아님. 신중의 "긍정적 측면"(꼼꼼한 확인, 리스크 점검)을 보여주는 선택지가 더 필요.

계획 Q16~Q18에서 신중 긍정 표현 문항 보강

문항 수 제한

15문항으로 3축을 측정하면 축당 문항 수가 적어 변별력 한계. 18문항으로 확장 시 축당 6문항으로 안정성 향상.

참고 DISPLAY_MAX=5는 15문항 기준 적정. 24문항 이상 시 상향 가능

사회적 바람직성

A선택지가 대부분 "바른 답"으로 보이는 경향. 선택지 순서 랜덤화(seededShuffle)로 부분 완화했으나, 내용 자체의 바람직성 차이는 완전히 해소되지 않음.

대응 Q3, Q7, Q8에서 사회적 바람직성 매칭 수정 완료

축별 최소 참여 보장 (해결됨)

각 선택지는 3축 중 1~3축에 점수를 부여합니다. 특정 축쌍에 점수를 주지 않는 "무관" 선택지가 존재할 수 있으나, "강제 참여" 문항(4개 선택지 모두 해당 축에 점수를 부여)을 축쌍당 3개 이상 확보하여 최소 참여를 보장합니다.

축쌍강제 참여 문항최소 참여 보장
원칙↔유연5개 (Q9, Q10, Q11, Q13, Q15)충족

8-1. 문항 설계 체크리스트

문항 추가·수정 시 반드시 확인해야 할 항목. 로직 레이어와 시각화 레이어를 모두 검증합니다.

#검증 항목기준검증 방법
1 축별 +1/-1 기회 균형 각 축쌍의 양방향 기회 차이 ≤ 3 기존 시뮬레이션
2 유형 분포 8유형 모두 출현율 5% 이상 100,000회 랜덤 시뮬레이션
3 사회적 바람직성 특정 선택지가 "정답"으로 보이지 않도록 문항 리뷰 프롬프트
4 축별 최소 참여 (강제 참여) 각 축쌍에 대해, 4개 선택지 모두 해당 축 점수를 부여하는 문항이 최소 3개 이상 선택지별 점수 매핑 전수 조사
5 레이더 차트 시각 검증 축쌍 양쪽 모두 1점(바닥)이 되는 응답 조합이 존재하지 않는지 확인 무관 선택지만 고른 시뮬레이션
#4 "강제 참여" 설명: 한 문항의 4개 선택지가 모두 해당 축에 점수를 주면, 그 문항에서는 어떤 선택을 해도 반드시 해당 축에 점수가 쌓입니다. 이런 문항이 축쌍당 최소 3개 있으면, 양쪽 합계 최소 3점이 보장되어 차트가 한쪽으로 쭈그러지지 않습니다.

9. 레이더 차트 (6축 단극)

결과 화면에서는 3축 양극이 아닌 6축 단극으로 레이더 차트를 표시합니다.

차트 축계산 방식원시 범위
원칙원칙(+1) 점수만 합산0 ~ max
유연유연(-1) 점수의 절대값 합산0 ~ max
투명투명(+1) 점수만 합산0 ~ max
신중신중(-1) 점수의 절대값 합산0 ~ max
독립독립(+1) 점수만 합산0 ~ max
협력협력(-1) 점수의 절대값 합산0 ~ max

표시 점수 정규화: 제곱근(√) 커브

이 테스트는 평가가 아닌 "긍정적 성향 발견"이 목적입니다. 사용자가 자신의 성향을 긍정적으로 인식하고, 청렴에 관심을 갖고 지키려는 동기를 얻도록 설계합니다. 이를 위해 원시 점수를 차트 표시 점수로 변환할 때 두 가지 보정을 적용합니다.

공식

score = round(MIN + √(raw / max) × (MAX − MIN))

DISPLAY_MIN = 1, DISPLAY_MAX = 5 → 표시 범위 [1, 5]

보정내용이유
바닥값 DISPLAY_MIN = 1
원시 점수 0이어도 최소 1점 표시		 대립축 구조상 한쪽이 0점은 불가피하나, "원칙 0점 = 원칙 없음"으로 오독되는 것을 방지
√ 커브 선형 대신 제곱근 함수로 매핑
중·상위 점수를 넓게 분포시킴		 대부분의 축이 3~5 범위에 표시되어 긍정적 인상을 주면서도 축 간 상대적 차이는 유지

선형 vs √ 커브 비교

raw / max선형 매핑√ 커브 (채택)차이
0%1점1점—
25%2점3점+1
50%3점4점+1
75%4점4점—
100%5점5점—
왜 √인가? log 커브는 축 간 차이가 거의 사라지고, x² 커브는 낮은 점수를 더 낮게 만들어 목적과 반대됩니다. √는 중간 강도로 끌어올려 긍정적 인상과 변별력을 동시에 확보하는 균형점입니다.
DISPLAY_MAX=5 근거: 15문항에서 축당 평균 2~4점. MAX=5일 때 √ 커브 적용 시 차트 60~100% 채움으로 시각적으로 적정. 문항 24개 이상으로 늘리면 MAX 상향 가능.
