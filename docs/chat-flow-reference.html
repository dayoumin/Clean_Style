3. API 모드 POST /api/analyze

초기 스타일 분석 — 테스트 직후 1회

입력설명

mode: question

이어서 질문 — 최대 8턴

입력설명
userContext질문 (필수, 500자)
history이전 대화 (선택)
summary요약 텍스트 (선택)
출력설명
answer일반 텍스트 (5-7문장)

temp: 0.7 / maxTokens: 600

mode: summarize

백그라운드 요약 — 4턴 도달 시 1회

입력설명
history첫 8개 메시지
styleKey유형 키
출력설명
summary3문장 이내 요약

temp: 0.3 / maxTokens: 200

System Prompt 분기 로직

조건프롬프트스타일 언급
첫 질문 isFirstTurn QA_SYSTEM_PROMPT 도입부 첫 문장에서 1회, 「꺾쇠」 사용
이어서 질문 hasHistory QA_SYSTEM_PROMPT_CONTINUE 유형명 절대 언급 금지
요약 있음 safeSummary CONTINUE + 요약 첨부 유형명 절대 언급 금지
isFirstTurn 판별: !hasHistory && !safeSummary
summary만 있고 history가 비어도 첫 턴이 아님 → 스타일 정보 재전송 방지

4. 채팅 데이터 흐름

fetchAnswer() 흐름

입력 검증
userContext.trim() 확인, 중복 요청 방지
히스토리 준비
요약 없음 → chatHistory 전체 전송
요약 있음 → chatHistory.slice(SUMMARIZE_AT).slice(-4) (최근 2턴만)
API 호출
POST /api/analyze { mode:'question', history, summary, userContext }
응답 처리
setAiAnswer → chatHistory에 [user, assistant] 추가 → slice(-16)
요약 트리거 체크
newHistory.length ≥ 8 && !chatSummary → triggerSummarize() (백그라운드)
localStorage 저장
useEffect로 chatHistory 변경 시 자동 저장 (updateChat)

9. 히스토리 (localStorage)

localStorage key: "integrity-history"

HistoryEntry {
  id:         string           // 자동 생성 UUID
  createdAt:  string           // ISO datetime
  styleKey:   string           // e.g. "principle-transparent-cooperative"
  styleName:  string           // e.g. "정의의 조율자"
  styleEmoji: string           // e.g. "⚖️"
  scores:     { principle, transparency, independence }
  answers:    number[]         // 15개 (0~3)
  chat:       ChatMessage[]    // { role, content }[]
}

MAX_ENTRIES = 10  (초과 시 가장 오래된 항목 삭제)
함수용도
getHistory()전체 목록 (최신 우선)
getHistoryEntry(id)단일 항목 조회
addHistoryEntry(params)새 항목 생성 (null if quota exceeded)
updateChat(id, chat)채팅 배열만 갱신
clearChat(id)채팅 비우기 (결과는 유지)
deleteHistoryEntry(id)항목 전체 삭제
