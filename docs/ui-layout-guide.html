4. Flex 컨테이너 구조

BottomSheet (flex-col, max-h-[85dvh] / sm:max-h-[70vh])
├── Header (shrink-0)
│   ├── "AI 맞춤 조언" 제목
│   └── ✕ 닫기 버튼
│
├── [답변 phase]
│   ├── 대화 영역 (flex-1 overflow-y-auto px-5 py-4)  ← SCROLL_AREA 상수
│   │   ├── ChatBubbles: 이전 히스토리 (chatHistory.slice(0,-2))
│   │   ├── ChatBubbles: 현재 질문 (user, 우측 정렬)
│   │   ├── 최신 답변 (좌측, 강조 border)
│   │   └── scrollAnchorRef (자동 스크롤 대상)
│   └── 하단 고정 (shrink-0 border-t px-5 py-3)
│       ├── 턴 안내 메시지 (조건부)
│       └── [새 질문] [이어서] 버튼 행
│
├── [로딩 phase]
│   └── 전체 스크롤 (flex-1 overflow-y-auto px-5 py-4)  ← SCROLL_AREA
│       ├── ChatBubbles: 히스토리 + 현재 질문
│       └── 스피너 + "답변을 작성하고 있어요..."
│
└── [입력 phase]
    ├── 히스토리 (flex-1 overflow-y-auto, 있을 때만 렌더)  ← SCROLL_AREA
    │   └── ChatBubbles: chatHistory 전체
    └── 하단 고정 (shrink-0, border-t 조건부)
        ├── textarea (3행, 500자 제한)
        ├── 에러 메시지 (조건부)
        ├── [질문하기] 버튼
        └── [대화 지우기] (히스토리 있을 때만)
SCROLL_AREA = 'flex-1 space-y-3 overflow-y-auto px-5 py-4'

8. 수정 시 체크리스트

변경 영역확인 사항
BottomSheet 높이/너비 변경모바일 85dvh + PC 70vh 둘 다 테스트, 키보드 올라올 때 확인
스크롤 영역 추가/변경SCROLL_AREA 상수 사용 여부, flex-1 + overflow-y-auto 쌍 확인
하단 고정 영역 추가shrink-0 + border-t 필수, 부모가 flex-col인지 확인
ChatBubbles 스타일 변경히스토리 버블 (ChatBubbles) + 최신 답변 (인라인) 두 곳 모두 수정
새 모달 phase 추가children으로 직접 전달 — BottomSheet는 래핑 div 없음, children이 flex-col 내부임을 인지
턴 제한 변경constants.ts 수정 후 chatMaxReached/chatLastTurn 계산 확인
브레이크포인트 추가현재 sm: 하나만 사용 — md: / lg: 추가 시 모달 내부도 대응 필요
BottomSheet 주의: children은 flex-col 컨테이너에 직접 들어갑니다 (래핑 div 없음). 따라서 children이 flex-1 / shrink-0을 직접 가져야 하며, 새 phase 추가 시 이 구조를 따라야 합니다.
