# Clean Style - 설계 결정 기록

## AI Q&A 대화 히스토리 설계

### 결정 사항
- **최대 5턴** (user+assistant 10메시지)까지 이어서 질문 가능
- 5턴 도달 시 "이어서" 버튼 숨기고 "대화가 길어져서 새로 시작할게요" 안내 후 "새 질문"만 표시
- 서버(`MAX_HISTORY_MESSAGES = 10`)와 클라이언트(`.slice(-10)`) 모두 동일하게 제한

### 대화 요약(Summarization) 미채택 이유
- 매 턴 요약(Rolling Summary): AI 호출 2배 (요약용 + 답변용) → 토큰 비용/응답 지연 증가
- 청크 요약(Chunk Summary): 역시 추가 AI 호출 필요
- 요약 품질에 따라 맥락 왜곡 우려
- 청렴 Q&A 특성상 긴 대화보다 짧고 독립적인 질문이 대부분 → 5턴 제한으로 충분
- ChatGPT, 16Personalities 등 주요 서비스도 컨텍스트 윈도우 기반 잘라내기 또는 턴 제한 사용

### 스타일 반영 전략
- **첫 질문**: 시스템 프롬프트(`QA_SYSTEM_PROMPT`)에서 "도입부에 스타일 한 번 언급" 지시, user 메시지에 스타일 정보 포함
- **이어서 질문**: 시스템 프롬프트(`QA_SYSTEM_PROMPT_CONTINUE`)에서 "스타일 반복 언급 금지" 지시, user 메시지에서 스타일 정보 제거
- 16Personalities 등 유명 서비스도 유형을 명시적으로 언급하되 과도하지 않게 하는 방식 채택

### 프롬프트 구조
- `QA_BASE`: 공통 역할/응답 원칙 (중복 방지)
- `QA_SYSTEM_PROMPT = QA_BASE + 첫 질문용 스타일 반영 원칙`
- `QA_SYSTEM_PROMPT_CONTINUE = QA_BASE + 이어서 질문용 스타일 반영 원칙`

### UI 구조
- 채팅 버블: 사용자(우측), AI(좌측) 배치
- `ChatBubbles` 공유 컴포넌트로 중복 제거
- 답변 화면 / 로딩 화면 / 입력 화면 모두 이전 대화 히스토리 표시
- 버튼: "새 질문" (히스토리 리셋) / "이어서" (히스토리 유지, 입력만 초기화)
