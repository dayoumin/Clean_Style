## 대화 축약 전략

### 방식

1. **턴 1~4**: 원본 대화 그대로 API에 전송
2. **턴 4 도달 시**: AI에게 "지금까지 대화를 3문장으로 요약해줘" 요청 → localStorage/IndexedDB에 저장
3. **턴 5~8**: 요약본 + 최근 2턴만 API에 전송

### 축약 시 비용 비교 (300명 × 2회 × 8턴)

| 시나리오 | 턴당 입력 토큰 | 총 비용 |
|----------|---------------|---------|
| 축약 없이 8턴 | 200→1400 (누적) | ~$1.45 |
| 축약 적용 8턴 | 200→600 (고정) | ~$0.80 |

### 축약의 주된 이점

- 비용 절감 (요약 호출 1회 추가 ~$0.05 감안해도 절감)
- **대화 품질 유지** — 긴 컨텍스트에서 AI 산만해지는 것 방지
- **재방문 시 대화 복원** 가능 (localStorage에 요약본 저장)
