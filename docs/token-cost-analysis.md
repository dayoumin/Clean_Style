# 토큰 & 비용 분석

## 모델 정보

- **플랫폼**: OpenRouter
- **모델 ID**: `x-ai/grok-4.1-fast`
- **기본 Temperature**: 0.7
- **토큰 단가**: 입력 $0.20/1M, 출력 $0.50/1M

## 대화 축약 전략 (구현됨)

### 전체 흐름

```
턴 1~4: 원본 대화 그대로 API에 전송 (chatHistory 누적)
턴 4 완료: 백그라운드로 요약 API 1회 호출 → React state에 저장
턴 5~8: [요약본 + 최근 2턴 + 현재 질문] API에 전송
```

### 턴별 API 전송 구성

| 턴 | 전송 내용 | 입력 토큰 (추정) |
|----|----------|-----------------|
| 1 | System + 스타일정보 + 질문 | ~800 |
| 2 | System + 턴1 히스토리 + 질문 | ~1,000 |
| 3 | System + 턴1~2 히스토리 + 질문 | ~1,200 |
| 4 | System + 턴1~3 히스토리 + 질문 | ~1,400 |
| — | **요약 호출**: 턴1~4 대화 → 3문장 요약 | ~880 (입력) |
| 5 | System + **요약** + 질문 | ~500 |
| 6 | System + **요약** + 턴5(2msg) + 질문 | ~700 |
| 7 | System + **요약** + 턴5~6(4msg) + 질문 | ~900 |
| 8 | System + **요약** + 턴6~7(4msg) + 질문 | ~900 |

### 잘라내기 vs 축약 비교

| | 잘라내기 (slice) | 축약 (summarize) |
|--|------------------|------------------|
| 오래된 대화 | **소실** | 요약으로 **맥락 유지** |
| "아까 말한 그거" | AI가 모름 | AI가 파악 가능 |
| 추가 API 호출 | 없음 | 요약 1회 (~$0.001) |
| 턴 5~8 입력 토큰 | 계속 누적 | 일정하게 유지 |
| 구현 복잡도 | 낮음 | 중간 |

### 요약본 저장 위치

- **React state** (`chatSummary`) — 세션 중에만 유지
- localStorage에 저장하지 않음
- 재방문 시 요약 없이 전체 히스토리로 동작 (MAX_HISTORY_MESSAGES 16개 cap)

## Fallback (API 실패 시)

API 호출 실패 시 **로컬 fallback 응답**이 반환됨 → 비용 $0.

- 8가지 스타일 유형별 사전 정의된 분석 결과 존재 (`api/analyze/route.ts`)
- 네트워크 오류, 타임아웃 등에서 자동 전환
- 요약 호출 실패 시 무시 — 전체 히스토리로 계속 진행

---

## 결론

8턴 + 축약 적용 시 비용 약 **$0.80** (300명×2회 기준).
축약의 주된 가치는 비용 절감보다 **대화 품질 유지** — 긴 컨텍스트에서 AI가 산만해지는 것을 방지.
