## 프로젝트 구조

- **형식**: 15문항, 4지선다, 강제선택
- **축**: 3축 양극 (원칙↔유연, 투명↔신중, 독립↔협력)
- **레이더 차트**: 6축 단극 (각 축 누적 점수 독립 표시), DISPLAY_MAX=5
- **유형**: 8가지 (2×2×2 조합)

## 축 간 점수 기회 불균형

6축 전환 후 축별 최대 가능 점수:

| 축 | 최대 가능 점수 | 비고 |
|---|---|---|
| 독립/협력 | 15 (모든 문항에서 기회) | |
| 원칙/유연 | 14 | |
| 투명 | 13 | |
| **신중** | **10** | Q1, Q3, Q6, Q11, Q13에 기회 없음 |

### 원칙+독립 공동 출현

15문항 중 9문항의 첫 번째 선택지가 `{principle:+1, independence:+1}`. 두 축이 독립적으로 측정되지 않음.

### 해결 방향

- 기존 문항 점수 수정: Q1c에만 신중 추가 가능 (효과 미미)
- **문항 3개 추가(Q16~Q18)가 권장**: 신중 기회 보강 + 원칙/독립 단독 점수 기회 확보. 기존 문항 무변경.

---

## DISPLAY_MAX=5 결정 근거

15문항 구조에서 축당 평균 2~4점. MAX=5일 때 차트 40~80% 채움으로 적정.
MAX=8이면 25~50%로 허전. 동적 스케일은 응답자 간 비교 불가.
문항 24개 이상으로 늘리면 MAX 상향 가능.

## 기존 결과 URL 하위 호환성

문항 추가 시:
- `computeSixAxisScores`는 `answers.forEach`로 순회 → 15개짜리 answers는 15문항만 처리
- `styleKey`, `p`, `t`, `i`는 URL에 하드코딩 → 유형 판정 불변
- 레이더 차트만 새 문항 미반영 (기존 15개 기반)
